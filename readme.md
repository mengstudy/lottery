# 双色球数据分析系统 - 使用指南

## 📖 系统简介

本系统是一个基于 Flask 的双色球数据分析网站，提供开奖数据查询、遗漏值统计、冷热号分析等功能。数据来源于中国福利彩票官方 API，确保准确性和及时性。

---

## 🚀 快速开始

### 环境要求

- Python 3.8+
- Windows / macOS / Linux

### 安装依赖

```bash
cd ssq_analyzer
pip install -r requirements.txt
```

### 启动应用

```bash
python start.py
```

### 访问网站

打开浏览器访问：**http://localhost:5000**

---

### 🖥️ 服务器部署（Linux）

#### 1. 创建 data 目录并设置权限

```bash
cd /home/lottery
mkdir -p data
chmod 755 data
```

#### 2. 自动修复权限（推荐）

```bash
python3 fix_permissions.py
```

#### 3. 启动应用

```bash
python3 start.py
```

📖 **详细部署指南**：请查看 [`部署问题修复指南.md`](部署问题修复指南.md)

## 📋 功能模块

### 1️⃣ 首页（最新数据）

**功能特点：**
- 显示最新一期开奖结果
- 实时遗漏值排行榜（Top 10）
- 冷热号分布统计
- 一键更新最新数据
- 计算遗漏次数表

**主要操作：**
- **立即更新**：抓取最新 1 期开奖结果（需口令验证）
- **🔢 计算遗漏次数表**：批量计算所有期次的遗漏次数（需口令验证）
- **查看详情**：点击期号进入详情分析页

---

### 2️⃣ 历史数据

**功能特点：**
- 倒序排列：最新数据在第 1 页
- 智能分页：最多显示 10 个页码，避免拥挤
- 页码跳转：直接输入页码快速定位
- 红球紧凑显示：不换行，美观整齐
- 地区悬浮提示：长文本省略，鼠标悬浮显示完整内容

**表格字段说明：**

| 字段 | 说明 | 示例 |
|------|------|------|
| 期号 | 彩票期号 | 2026026 |
| 日期 | 开奖日期 | 2026-03-08 |
| 红球 | 6 个红色球号码 | 03 08 11 12 15 22 |
| 蓝球 | 1 个蓝色球号码 | 16 |
| 奇偶比 | 红球奇数：偶数 | 3:3 |
| 最大遗漏 | 红球最大遗漏值 | 18 |
| 蓝球遗漏 | 当前蓝球遗漏次数 | 5 |
| 红球遗漏详情 | 开出的 6 个红球的遗漏值（排序） | 0-1-2-7-9-15 |
| 中奖地区| 一等奖中奖地区| 北京市、天津市...（悬浮查看） |

**主要操作：**
- **更新最新数据**：抓取最新 1 期开奖结果（需口令验证）
- **🔄 更新所有历史数据**：批量抓取所有历史数据（需口令验证）
- **上一页/下一页**：翻页浏览
- **页码跳转**：输入页码后点击“跳转”
- **查看详情**：点击期号进入详情分析页

---

### 3️⃣ 详情分析

**功能特点：**
- 完整的开奖信息展示
- 所有红球的遗漏次数分组（全部 33 个红球）
- 上一期遗漏分组对比（标记本期开出号码）
- 遗漏值可视化图表
- 冷热号分析

**遗漏次数分组：**

| 分组 | 遗漏次数 | 颜色 | 说明 |
|------|---------|------|------|
| A 组 | 0 次 | 🔴 红色 | 本期开出（热号） |
| B 组 | 1-5 次 | 🟡 黄色 | 温热号码 |
| C 组 | 6-8 次 | 🔵 蓝色 | 较冷号码 |
| D 组 | ≥9 次 | 🟢 青色 | 极冷号码 |

**主要操作：**
- **返回历史列表**：点击"返回历史数据列表"
- **查看其他期号**：在历史列表中切换

## 🎯 核心特性

### ✅ 数据管理

- **倒序显示**：最新数据在第 1 页，符合用户习惯
- **立即保存**：每抓取 1 页立即保存，防止数据丢失
- **断点续传**：中途中断不影响已保存的数据
- **增量更新**：只抓取缺失的期次，提高效率
- **官方数据源**：使用中国福彩官方 API，准确可靠

### ✅ 界面优化

- **响应式设计**：适配不同屏幕尺寸
- **智能分页**：超过 10 页时精简显示
- **悬浮提示**：长文本字段支持鼠标悬浮查看
- **视觉标记**：冷热号用颜色区分，一目了然
- **紧凑布局**：红球不换行，节省空间

### ✅ 性能优化

- **数据缓存**：数据库查询优化，加载迅速
- **按需计算**：遗漏值预先计算，实时展示
- **异步更新**：后台更新数据，不阻塞界面

## 📊 数据更新

### 手动更新

#### 方式 1：更新最新数据（推荐）
1. 访问首页
2. 点击 **"更新最新数据"** 按钮
3. 系统自动抓取最新 1 期并保存
4. 刷新页面查看新数据

#### 方式 2：更新所有历史数据
1. 访问首页
2. 点击 **"🔄 更新所有历史数据"** 按钮
3. 系统抓取全部历史数据（约 3000 期，需 2-3 分钟）
4. 完成后刷新页面

### 自动更新

系统配置了定时任务：
- **更新时间**：每周二、四、日 21:30（开奖时间）
- **更新内容**：自动抓取最新一期数据
- **日志记录**：更新日志保存在 `crawler.log`

### 技术栈

| 层次 | 技术 | 说明 |

| 后端框架 | Flask 3.0 | 轻量级 Web 框架 |

## 🔧 常见问题

### Q1: 数据保存在哪里？
**A:** 数据库文件位于 `data/lottery.db`，SQLite 格式，可用数据库工具查看。

### Q2: 如何查看更新日志？
**A:** 日志实时显示在终端窗口，同时保存在 `crawler.log` 文件中。

### Q3: 分页太多找不到某一页？
**A:** 使用页面底部的"页码跳转"功能，直接输入页码后点击"跳转"。

### Q4: 中奖地区文字太长看不清？
**A:** 鼠标悬浮在文字上即可看到完整内容（tooltip 提示框）。

### Q5: 红球遗漏详情如何解读？
**A:** 显示的是开出的 6 个红球当前的遗漏次数，按从小到大排序，例如 `0-1-2-7-9-15`。

### Q6: 如何停止应用？
**A:** 在终端窗口按 `Ctrl+C` 即可停止服务。

### Q7: 数据多久更新一次？
**A:** 
- 手动更新：随时可以更新
- 自动更新：每周二、四、日 21:30（开奖后）

### Q8: 为什么有些期次没有中奖地区？
**A:** 部分早期数据官方 API 未提供地区信息，显示为"-"。

## 🛠️ 故障排除

### 问题 1：无法启动应用
**解决方案：**
```bash
# 检查 Python 版本
python --version  # 需要 3.8+

# 重新安装依赖
pip install -r requirements.txt --upgrade
```

### 问题 2：数据库错误
**解决方案：**
```bash
# 备份数据库（可选）
cp data/lottery.db data/lottery.db.bak

# 删除数据库重新初始化
rm data/lottery.db
python start.py
```

### 问题 3：端口被占用
**解决方案：**
```bash
# 修改 app.py 中的端口号
app.run(host='0.0.0.0', port=5001)  # 改为 5001

# 或者关闭占用端口的程序
```

### 问题 4：数据抓取失败
**解决方案：**
1. 检查网络连接
2. 检查防火墙设置
3. 查看 `crawler.log` 了解详细错误
4. 稍后重试
