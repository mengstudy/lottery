1. 项目背景与核心目标
一句话概述： 这是一个为彩票爱好者或数据分析师设计的双色球历史数据统计分析工具。它能自动抓取官方开奖数据，并基于遗漏次数这一核心指标，对每期开奖号码进行深度剖析和可视化展示。

核心痛点： 手动统计遗漏值、奇偶比等数据非常繁琐且易出错；难以直观地看出冷热号分布。

成功指标： 用户访问网站即可获取最新数据，并能清晰地看到每期号码的遗漏特征（如：上期开出的6个红球分别遗漏了多少期？当前哪些号码是冷号？）。

2. 技术栈规范
后端框架： Flask (轻量级，适合本项目) 或 Django (如需后续扩展复杂功能) 。

数据库： SQLite (轻便，无需单独安装数据库服务，数据可存为本地文件) 。

数据抓取： Python requests + BeautifulSoup4 (解析HTML) 。需注意官网可能反爬，可配置User-Agent和抓取间隔 。

数据分析： Pandas (用于复杂的数据透视和过滤)，collections.Counter (用于频率统计) 。

前端可视化： ECharts 或 Chart.js (用于绘制遗漏折线图、冷热号柱状图) 。

定时任务： 使用 APScheduler 或系统Crontab，定期（如每天开奖后）执行爬虫脚本更新数据库 。

5. 核心功能模块与逻辑
5.1 数据抓取模块 (crawler/crawler.py)
目标源： 首选 https://www.cwl.gov.cn/ygkj/wqkjgg/ssq/。若官网抓取失败，备选方案可为 datachart.500.com/ssq/，其数据结构相对稳定 。

策略：

首次全量抓取： 循环遍历历史页面，获取所有历史数据并存入数据库。
增量更新： 定时任务运行时，只抓取最新的几期数据，与数据库中最新一期对比，若不存在则插入 。
异常处理： 需处理网络超时、验证码（如出现）、数据解析失败等情况，并记录日志。

5.2 遗漏值计算模块 (analyzer/stats_calculator.py)
这是项目的数学核心。需要先定义“遗漏值”的概念：指某个号码自上次开出后，未出现的期数 。

算法逻辑：

从数据库按期号升序取出所有开奖记录。
初始化两个字典：red_missing_count (键：红球01-33，值：当前遗漏期数) 和 blue_missing_count (键：蓝球01-16，值：当前遗漏期数)。
遍历每一期记录：
对于红球 01 到 33：
如果该期开奖红球列表包含当前号码，则该号码的遗漏值重置为 0。
否则，该号码的遗漏值 += 1。
蓝球同理。
将每一期计算出的每个号码的遗漏值和该期开奖号码的遗漏值存储起来（可以设计另一个数据表，或直接在内存中用于生成视图）。
5.3 数据分析与统计模块
在计算出遗漏值的基础上，实现以下功能：

每期号码遗漏分析： 对于某一期开出的号码（如红球 [05, 12, 18, 23, 30, 33]，蓝球 07），查看它们在开奖前的遗漏次数分别是多少。这能反映这期号码是“热号”（遗漏少）还是“冷号”（遗漏多）。

奇偶比计算： 统计每期开出的6个红球中，奇数与偶数的数量比（如 3:3, 4:2）。

冷热号分组： 以当前最新一期开奖后为时间点，将所有红球和蓝球按遗漏次数分组 ：

热号 (0次遗漏): 最近一期刚刚开出的号码。

温号 (1-5次遗漏): 遗漏次数较少的号码。

冷号 (6-10次遗漏): 遗漏次数适中的号码。

大冷号 (>10次遗漏): 长期未开出的号码，也是分析的重点。

6. 页面功能与交互设计
6.1 首页/最新数据看板 (/)
功能：

展示最新一期的开奖结果（期号、日期、红球、蓝球）。

核心看板：实时遗漏值排行榜。以表格或卡片形式，展示当前所有红球和蓝球的遗漏次数，并按遗漏值从高到低排序，让“大冷号”一目了然。

冷热号分组展示区： 直接将红球01-33和蓝球01-16按上述“冷热号分组”逻辑，分为几个区块展示，不同区块用不同颜色背景区分。

6.2 历史数据与分析页面 (/history 或 /analysis)
功能 A：历史开奖列表

以分页表格形式展示历史开奖记录。

关键列： 期号、开奖日期、红球号码、蓝球号码、奇偶比、红球最大遗漏值（指本期的6个红球中，最大的遗漏次数是多少）、蓝球遗漏值。

功能 B：期次详情分析 (点击某期进入)

本期间号码遗漏详情： 针对这一期的7个开奖号码，列表展示每个号码在开奖前的具体遗漏次数。

示例：

期号：2024101
开奖号码：红 05, 12, 18, 23, 30, 33 | 蓝 07
遗漏分析：

红球 05 (遗漏 3期)

红球 12 (遗漏 1期)

...

蓝球 07 (遗漏 5期)

功能 C：可视化图表

号码遗漏走势图： 提供一个下拉框选择具体号码（如红球01），生成该号码的遗漏值随时间（期数）变化的折线图 。

冷热号分布图： 使用柱状图展示所有号码的出现频率或当前遗漏值分布。

注：建议将定时任务设置在官方开奖时间（每周二、四、日晚）之后运行。

8. 非功能性需求
数据准确性： 爬虫模块必须有重试机制，且数据入库前要进行基本校验（如红球数量是否为6，且均在1-33范围内）。

性能： 首页的冷热号分组和遗漏值排行榜，应考虑在数据更新时（如每日）预计算并缓存结果，而不是每次用户访问时都全量计算，以提升页面加载速度。

易用性： 页面需提供明确的刷新或“立即更新”按钮，供高级用户手动触发数据更新。
