# 遗漏次数表功能说明

## 📊 功能概述

系统新增了两个数据库表来**预先存储**每一期所有号码的遗漏次数，避免每次重复计算。

---

## 🗄️ 数据表结构

### 1. 红球遗漏次数表 (red_ball_missing)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| issue | TEXT | 期号（唯一键） |
| num_01 ~ num_33 | INTEGER | 1-33 号红球的遗漏次数 |
| updated_at | TIMESTAMP | 更新时间 |

**示例数据：**
```
issue: 2026026
num_01: 15, num_02: 8, num_03: 0, num_04: 23, ...
(开出的号码遗漏值为 0，未开出的累加)
```

### 2. 蓝球遗漏次数表 (blue_ball_missing)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| issue | TEXT | 期号（唯一键） |
| num_01 ~ num_16 | INTEGER | 1-16 号蓝球的遗漏次数 |
| updated_at | TIMESTAMP | 更新时间 |

## 🔧 使用方法

### 方式 1：通过首页按钮（推荐）

1. 访问网站首页 **http://localhost:5000**
2. 点击页面顶部的 **"🔢 计算遗漏次数表"** 按钮
3. 系统会弹出提示框，点击确定开始计算
4. 等待计算完成（约 30-60 秒）
5. 计算完成后自动刷新页面

**特点**：
- ✅ 一键操作，无需手动执行命令
- ✅ 实时显示进度和结果
- ✅ 自动验证计算结果
- ✅ 友好的用户界面

### 方式 2：单独脚本计算

如果 API 不可用，可以创建独立脚本：

```python
# calculate_missing.py
from database.db_manager import db_manager
from analyzer.stats_calculator import calculator

def calculate_all():
   print("开始计算遗漏次数表...")
    
    # 获取所有数据（升序）
    all_results = db_manager.get_all_results(order_by='draw_date')
    
    if not all_results:
       print("没有开奖数据！")
       return
    
    total = len(all_results)
   print(f"共 {total} 期数据")
    
    # 清空旧数据
    db_manager.delete_all_red_ball_missing()
    db_manager.delete_all_blue_ball_missing()
   print("已清空旧数据")
    
    # 逐期计算
    for idx, result in enumerate(all_results):
       issue = result['issue']
        
        # 计算该期遗漏
       red_missing, blue_missing = calculator.calculate_all_missing_for_issue(all_results, idx)
        
        # 保存
        db_manager.insert_red_ball_missing(issue, red_missing)
        db_manager.insert_blue_ball_missing(issue, blue_missing)
        
        if (idx +1) % 100 == 0:
           print(f"已处理 {idx+1}/{total} 期")
    
   print(f"✅ 计算完成！共处理 {total} 期")

if __name__ == "__main__":
    calculate_all()
```

运行：
```bash
cd ssq_analyzer
python calculate_missing.py
```

## 📈 数据查询

### 查询某期的红球遗漏

```python
from database.db_manager import db_manager

missing = db_manager.get_red_ball_missing('2026026')
if missing:
   print(f"2026026 期红球遗漏:")
    for num in range(1, 34):
        field = f'num_{num:02d}'
       print(f"{num:02d}号：{missing[field]}次")
```

## 🎯 优势对比

### 优化前
- ❌ 每次查看历史列表都要重新计算
- ❌ 查看详情页也要重新计算
- ❌ 大量重复计算，浪费资源
- ❌ 响应速度慢

### 优化后
- ✅ 一次性计算，永久存储
- ✅ 查询时直接读取，无需计算
- ✅ 大幅提升性能
- ✅ 支持快速筛选和排序

## ⚠️ 注意事项

1. **数据更新后需要重新计算**
   - 新增开奖数据后，建议重新运行"一键计算"
   - 或者只计算新增的期次（可优化）

2. **磁盘空间**
   - 每个表约占用几 MB 空间
   - 完全可接受

3. **计算时机**
   - 建议在系统空闲时计算
   - 避免在用户访问高峰期计算
